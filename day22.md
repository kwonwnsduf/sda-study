#인덱스 기초
## 인덱스란?
인덱스 = DB에서 검색 속도를 빠르게 하기 위한 자료구조

	책으로 치면 목차

	인덱스 없으면 → 풀 테이블 스캔
	
	인덱스 있으면 → 필요한 위치로 바로 점프
    	
    SELECT 성능 ↑

	INSERT / UPDATE / DELETE 성능 ↓ 
    
## 인덱스 구조 — B-Tree
왜 B-Tree?
	
    정렬된 상태 유지

	범위 검색 가능

	균형 트리 → 깊이가 얕음 → 디스크 I/O 최소화
    
>  루트
 ├── 중간 노드
 │    ├── 리프 노드
 │    └── 리프 노드
 └── 중간 노드
루트
 ├── 중간 노드
 │    ├── 리프 노드
 │    └── 리프 노드
 └── 중간 노드

	인덱스 컬럼 값

	실제 row 위치(pointer)
    
## 인덱스를 쓰는 경우
	WHERE 조건에 자주 등장

	카디널리티 높은 컬럼 (값 종류 많음 → id, email, user_id)

	JOIN 조건

	ORDER BY

	GROUP BY
    
## 인덱스를 쓰면 안 되는 경우
	값 종류 거의 없음 (성별, Y/N, 상태값 )

	데이터 적은 테이블

	자주 UPDATE 되는 컬럼
   
## 인데스 단점
> | 작업     | 영향         |
| ------ | ---------- |
| INSERT | 인덱스도 같이 정렬 |
| UPDATE | 인덱스 재정렬    |
| DELETE | 인덱스 삭제     |

쓰기 많은 테이블 = 인덱스 최소화

## 인덱스 종류
1. PRIMARY KEY

	자동 인덱스

	클러스터드 인덱스

2. UNIQUE INDEX

	중복 방지 + 검색 속도

3.	일반 INDEX
> CREATE INDEX idx_user_id ON orders(user_id);
