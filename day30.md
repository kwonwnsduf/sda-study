#식별자
## 식별자란?
식별자 = 엔터티 내의 각 인스턴스를 유일하게 구분하는 속성

## 식별자 조건
| 조건  | 의미        |
| --- | --------- |
| 유일성 | 중복 X     |
| 최소성 | 꼭 필요한 것만  |
| 불변성 | 값이 변하지 않음 |
| 존재성 | NULL X   |

## 식별자 종류
### 기본 식별자(PK)
테이블의 대표 식별자

엔터티당 1개

### 보조 식별자
PK는 아니지만 유일성 만족

ex) 회원ID, 이메일(보조 식별자)

### 외부 식별자(FK)
다른 엔터티의 식별자를 가져온 것

## 본질 식별자 vs 인조 식별자
### 본질 식별자
업무적으로 의미 있음/현실 세계 값

단점: 변경 가능성 있고 개인정보/정책 이슈가 있다

### 인조 식별자
시스템이 만든 ID
의미 없음

## 단일 식별자 vs 복합 식별자
### 단일 식별자
속성 1개, 관리 쉬움, JOIN 단순

### 복합 식별자
속성 2개 이상, 주로 관계 엔티ㅣ에서 사용

## 식별자 선택 기준
1. 불변성
2. 최소성
3. 단순성
4. 의미 배제
