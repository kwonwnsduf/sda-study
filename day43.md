## RDS

RDS = AWS가 운영(백업·패치·복구)을 대신해주는 관리형 DB

## Multi-AZ — “DB 고가용성의 핵심”

Multi-AZ = 장애 발생 시 자동으로 다른 AZ의 대기 DB로 전환하는 구조

읽기 성능 향상 혹은 사용자가 두 DB에 직접 접속하는 것이 아닌 장애 대응에 관한 이야기

Primary DB: 실제로 쓰는 DB (AZ-A)

Standby DB: 대기용 DB (AZ-B)

데이터는 동기 복제

장애 시 자동 Failover

애플리케이션은 엔드포인트 그대로

## Multi-AZ vs Read Replica

| 구분   | Multi-AZ  | Read Replica |
| ---- | --------- | ------------ |
| 목적   | 장애 대응     | 읽기 성능        |
| 복제   | 동기        | 비동기          |
| 접속   | 하나의 엔드포인트 | 각자 엔드포인트     |
| 장애 시 | 자동 전환     | 수동 승격        |

## 자동백업

자동 백업 = RDS가 매일 자동으로 하는 백업 + PITR

자동/보존 기간 설정(1~35일)/ PITR 가능/ DB삭제 시 백업도 같이 삭제

PITR: 시간 지정 복구

## 스냅샷

사용자가 직접 찍는 수동 백업

수동, 보존 기간 x, DB 삭제해도 스냅샷은 남는다/ 다른 리전으로 복사 가능
