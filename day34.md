# 데이터 무결성
## 무결성이란
데이터가 정확하고, 일관되고, 신뢰 가능한 상태

## 무결성 종류

### 개체 무결성
PK는 NULL x
PK는 중복 x

### 참조 무결성
FK는 부모 테이블 PK에 존재해야

부모 삭제 시 자식 처리 규칙 존재

옵션:
	ON DELETE CASCADE: 부모 데이터가 삭제되면, 그와 연결된 자식 데이터도 같이 삭제된다.

	ON DELETE SET NULL: 부모 데이터가 삭제되면, 자식 데이터의 외래키(Foreign Key) 값을 						  NULL로 변경한다.

	RESTRICT: 자식 데이터가 하나라도 남아 있다면, 부모 데이터를 삭제하지 못하도록 막는다.
	
### 도메인 무결성
컬럼 값의 범위 / 형식 / 타입 제한

ex) age INT CHECK (age >= 0)
status VARCHAR(10) CHECK (status IN ('Y','N'))

# 데이터 표준 & 데이터 품질

## 데이터 표준
 목적: 이름, 정의, 형식 통일
 
 구성 요소

	표준 용어 (Customer, Order)

	표준 도메인 (Y/N, 코드값)

	표준 코드

“데이터 표준의 목적은?”

	일관성 확보

	의사소통 개선

	데이터 품질 향상
  
## 데이터 품질 요소

| 요소  | 의미              |
| --- | --------------- |
| 정확성 | 값이 사실과 맞는가      |
| 일관성 | 같은 의미 = 같은 값    |
| 유효성 | 허용 범위 내 값       |
| 적시성 | 시점이 적절한가        |
| 완전성 | NULL 없이 잘 채워졌는가 |

# 메타데이터 & 데이터 관리

## 메타 데이터
데이터를 설명하는 데이터

ex) 테이블 정의, 컬럼 타입, 제약조건, 인덱스 정보
 
## 데이터 관리 체계
데이터 거버넌스: 조직 전체에서 데이터를 정확하고 안전하게 사용하기 위해 만든 총체적인 관리 시스템

데이터 관리자: 거버넌스 원칙에 따라 현장에서 데이터를 실제로 관리하는 '데이터 집사'

데이터 표준 관리: 조직 내에서 쓰는 용어, 형식, 단위를 하나로 통일하는 작업
