#식별 관계 vs 비식별 관계
> 자식의 PK에 부모의 PK가 포함되면 → 식별 관계
자식의 PK에 부모의 PK가 포함되지 않으면 → 비식별 관계

## 식별관계
부모 엔터티의 식별자가 자식 엔터티의 식별자(PK)에 포함되는 관계

자식 PK = 부모 PK + 자식 고유값

부모 없이는 자식 존재 불가

존재 종속 관계

ERD 표기: 실선

ex) 주문-주문상세   Ordertem Pk구성(order_id,item_seq)

## 비식별 관계
부모의 식별자가 자식 엔터티의 일반 속성(FK)으로만 존재하는 관계

자식 PK = 자식 고유 PK

부모 PK는 FK일 뿐

식별 독립

ERD 표기: 점선

ex) 회원-주문 Order 컬럼(order_id(pk)/member_id(fk))

## 두 관계를 표로 구분
| 구분    | 식별 관계     | 비식별 관계  |
| ----- | --------- | ------- |
| 부모 PK | 자식 PK에 포함 | FK로만 존재 |
| 자식 식별 | 부모에 의존    | 독립      |
| ERD 선 | 실선        | 점선      |
| 사용 상황 | 약한 엔터티    | 일반 엔터티  |

## 언제 무엇을 쓸까?
### 식별 관계를 쓰는 경우
자식이 독립적으로 의미 없음

부모 없이는 존재 불가

관계 엔터티 / 상세 엔터티

ex) 주문-주문상세 || 학생-수강

### 비식별 관계를 쓰는 경우
자식이 독립 식별 가능

관리·확장 유리

PK 단순

ex) 회원–주문 || 부서–사원
