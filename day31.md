#관계
## 관계란?
관계 = 엔터티 간의 업무적 연관성

## 관계의 요소
| 요소              | 의미              |
| --------------- | --------------- |
| 관계명             | 두 엔터티의 의미       |
| 차수(Cardinality) | 1:1 / 1:N / N:M |
| 선택성(Optional)   | 필수 / 선택         |

## 관계명
관계명은 동사 형태로 읽는다

ex) 회원 은 주문을 한다/주문 은 회원에 의해 생성된다

## 관계 차수
1:1/1:N/N:M

N:M관계는 중간 엔티티 추가

## 선택 관계 vs 필수 관계
선택성 = “존재해야 하냐?”

필수 관계: 반드시 상대 엔터티 존재

예)주문은 반드시 회원이 있어야 함

선택 관계: 없어도 됨

예) 회원은 주문이 없을 수도 있음

## 표기 해석

| 표기   | 의미   |        |       |
| ---- | ---- | ------ | ----- |
| `    |      | `      | 정확히 1 |
| `O   | `    | 0 또는 1 |       |
| `    | <`   | 1 이상   |       |
| `O<` | 0 이상 |        |       |

ex)회원  || ───── O<  주문

읽기: 한 주문은 반드시 한 회원에 속한다.
	 한 회원은 주문이 없을 수도, 여러 개일 수도 있다.
